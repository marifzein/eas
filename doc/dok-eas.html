Model Relasi Penting Catatan untuk Developer

Untuk lebih detail field-field masing-masing model bisa dilihat di bisa dilihat di eas/evaluasi/models.py :

from django.contrib.auth.models import User

class Santri(models.Model):
  
  user = models.OneToOneField(User, on_delete=models.CASCADE, null=True, blank=True)
  hp = models.CharField(max_length=20)
  gender=models.CharField(choices=[('L','Laki-laki'),('K','Perempuan')], max_length=1,default='L')
  tgl_lahir=models.DateField()
  tahun_penerimaan=models.SmallIntegerField()

  def __str__(self):
    return f"Santri ID: {self.pk}"

class Soal(models.Model):
  pertanyaan = models.CharField(max_length=255)
  jawaban_benar = models.CharField(max_length=1)#a,b,c,d

  def __str__(self):
    return self.pertanyaan
  
class PilihanJawaban(models.Model):
  soal=models.ForeignKey(Soal,related_name="pilihan",on_delete=models.CASCADE)
  kode=models.CharField(max_length=1)#a,b,c,d
  teks=models.CharField(max_length=255)

  def __str__(self):
    return f"{self.kode}. {self.teks}"
  
class Nilai(models.Model):
  santri = models.ForeignKey(Santri,related_name="nilai_santri",on_delete=models.CASCADE)
  nilai=models.DecimalField(max_digits=5,decimal_places=2)

  def __str__(self):
    return str(self.nilai)

class Evaluasi(models.Model):
  santri =models.ForeignKey(Santri,related_name="jawaban_santri",on_delete=models.CASCADE)
  soal =models.ForeignKey(Soal,related_name="jwbn_santri",on_delete=models.CASCADE)
  jawaban_santri=models.CharField(max_length=1)#a,b,c,d

  def __str__(self):
    return self.jawaban_santri

Aplikasi EAS adalah kuis sederhana berbasis Django dengan fokus pada otentikasi santri melalui NIP (ditempatkan di field username Django User), serta form Evaluasi untuk mengerjakan soal dan form Peringkat untuk menampilkan peringkat peserta evaluasi.

2.2.b. Function Logout

@login_required
def  logoutView(request):
	context = {
	'page_title' : 'Homepage',
	'page_text' : 'Ahlan wa Sahlan'
}
	logout(request)
	return  render(request, 'home.html', context)
  • Fungsi :
    untuk logout dari system otentikasi django, sehingga user tidak lagi punya akses ke aplikasi. Fungsi ini menggunakan decorator @login_required , mewajibkan agar user harus login untuk bisa logout, untuk menghindari bypass dengan langsung akses ke url logout.
    Tampilan akan menggunakan template home.html

2.2.c Function login

Dibagi menjadi 2 berdasarkan request method POST dan GET.

santri_obj = Santri.objects.get(user=user_info)

statEva = Nilai.objects.filter(santri=santri_obj).exists()

user = authenticate(request, username=username_login, password=username_password)

Jika valid akan dicek apakah user ada di model Santri , dan apakah sudah evaluasi :
